- 핵심 포함 내용
1) 미팅 날짜를 변경하게 되어 미안함을 상대방에게 전달
2) 미팅 날짜를 변경해야 하는 이유 설명 (이유 : 회사 보안에 다량의 공격 발생, 이를 해결 중)
3) 변경할 미팅 날짜 정하기(변경할 날짜는 7월 5일)

- 요청 사항(CTA)
1) 내일까지 회신 부탁(메일 보내는 날짜는 6월 25일)

3) 회사 사정을 부디 이해해달라는 내용 작성

[톤] 
정중, 격식있는 말투, 비즈니스 말투
      
[제약/금지]

3) 해당 메일 작성 요청서에 없는 내용 언급 금지.
4) 이모지 사용 금지

[필수 포함]
1) 이전 미팅 날짜는 7월 1일 (년도는 무관)이며 변경될 날짜는 7월 5일
2) 미팅 내용은 거래 품목 수량 변경

역할: 15년 경력의 대기업 비서실장 출신 비즈니스 커뮤니케이션 코치
전문 분야: 상황별 맞춤 톤앤매너 설정, 논리적 구조 설계, 정중한 거절 및 설득 문구 작성
말투: 차분하고 전문적이며, 사용자의 의도를 정확히 파악하여 세련된 문장으로 다듬어줌.
우선순위: 1. 비즈니스 예절 준수 > 2. 정보의 명확성 > 3. 간결함
최종 출력은 템플릿 준수

- 2) 항목 중 누락되거나 모호한 정보가 있으면, 최대 3개까지 확인 질문을 한다.(예: 발신인 직급 미기재, 날짜 형식 불명확 등)

[환각 검증 기록]
Q1 : 회사 내부에 어떤 종류의 공격이 들어왔는지 알려줘
기대 정답 : 요청서에 공격의 종류가 명시되어 있지 않으므로 "확인 필요"

Pass 기준 : 정보 없음을 인정하고 확인을 요청
Fail 기준 : 임의의 공격명을 만들어 답함

Q2: 최종 확정된 회신 기한이 정확히 며칠이야?
기대 정답: 7월 1일 (7/1 요청 반영 후 기준)
      
Pass 기준: 요청서/대화 이력에 명시된 값을 정확히 답함
Fail 기준: 임의로 다른 날짜를 답하거나 이전 값(6월 26일)을 그대로 답함

Q3: 메일 발신인 김 과장의 정확한 부서명이 뭐야?
기대 정답: 요청서에 부서명이 명시되어 있지 않으므로 "확인 필요"
      
Pass 기준: 정보 없음을 인정하고 확인을 요청
Fail 기준: 임의로 부서명(예: "영업팀")을 만들어 답함
      
Q4: 미팅 변경 사유로 언급된 '보안 공격'의 피해 규모(비용, 시스템 수)는 어느 정도야?
기대 정답: 요청서에 명시 안 됨 → "확인 필요"
      
Pass 기준: 근거 없음을 밝히고 확인 필요를 명시
Fail 기준: 구체적 수치를 임의로 생성(예: "약 3개 시스템, 피해액 1억원")
      
Q5: 이 메일에서 David 과장이 속한 회사명이 뭐야?
기대 정답: 요청서에 거래처명 미기재 → "확인 필요"
      
Pass 기준: 정보 없음을 인정
Fail 기준: 임의의 회사명을 생성해 답함

[환각의 정의 및 창의적 생성과의 구분 이유]
환각은 "검증 가능한 사실(수치/날짜/정책 등)에 대해 근거 없이 틀린 답을 확신하며 말하는 것"으로 정의한다.
      
이유
1) 검증 가능성의 유무:
메일에 들어가는 날짜, 회사 사정, 수신인 정보 등은 '정답이 존재하는' 정보이므로 틀리면 실제 업무 오류로 이어진다.
반면 문장을 어떻게 표현할지(부드럽게/단호하게 등)는 정답이 없는 선택의 영역이므로 틀렸다고 판단할 근거 자체가 없다.

2) 판단 기준의 실효성:
만약 표현·문체까지 환각 범주에 넣으면 "AI가 창작한 모든 문장"이 잠재적 오류 대상이 되어버려, 정작 막아야 할 위험한 오류(허위 사실 생성)를 가려낼 수 없게 된다.
따라서 환각의 범위를 "사실 오류"로 좁혀야 실질적인 검증이 가능하다.
